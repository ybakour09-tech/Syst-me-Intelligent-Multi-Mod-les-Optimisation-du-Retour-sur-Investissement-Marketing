================================================================================
RÉSUMÉ COMPLET DU PROJET : OPTIMISATION DU RETOUR SUR INVESTISSEMENT MARKETING
================================================================================

Ce document retrace chronologiquement toutes les actions, analyses et décisions prises lors de notre session de travail sur le jeu de données "Dummy Data HSS".

1. INITIALISATION ET RÉCUPÉRATION DES DONNÉES
--------------------------------------------------------------------------------
- Examen de l'espace de travail Git local et identification du script `import.py`.
- Installation de la bibliothèque `kagglehub` via pip.
- Exécution du script pour télécharger le jeu de données "Dummy Data HSS.csv" depuis Kaggle.
- Déplacement du jeu de données brut dans le sous-dossier `data/` pour structurer le projet.

2. ANALYSE EXPLORATOIRE DES DONNÉES (EDA)
--------------------------------------------------------------------------------
- Création de scripts d'analyse (`analyze.py`, `visualize.py` puis `dist_check.py`) pour explorer le fichier brut.
- Constats initiaux :
  * Les types d'influenceurs (Mega, Macro, Micro, Nano) sont parfaitement équilibrés (~25% chacun).
  * Présence de très rares valeurs manquantes (NaNs) sur l'ensemble des colonnes.
  * Forte relation linéaire identifiée visuellement et statistiquement entre les budgets publicitaires et les ventes (Sales).

3. NETTOYAGE ET PRÉTRAITEMENT (PREPROCESSING)
--------------------------------------------------------------------------------
- Création du script `preprocess.py` pour nettoyer les données :
  * Suppression de la colonne `Influencer` (non pertinente pour l'étude).
  * Suppression des quelques lignes contenant des valeurs manquantes (impact négligeable : ~0.56% des données).
  * Détection et suppression des valeurs aberrantes (outliers) via la méthode de l'écart interquartile (IQR), supprimant 29 lignes extrêmes.
- Sauvegarde d'un jeu de données intermédiaire propre : `data/Clean_Data_HSS.csv`.

4. SÉLECTION DE CARACTÉRISTIQUES (FEATURE SELECTION) & MULTICOLINÉARITÉ
--------------------------------------------------------------------------------
- Calcul du Facteur d'Inflation de la Variance (VIF) révélant une très forte multicolinéarité entre la `TV` et la `Radio` (VIF > 12).
- Lancement d'une étude combinatoire exhaustive (`combinatorial_selection.py`) testant toutes les combinaisons possibles de variables (TV, Radio, Social Media) sur des modèles de Random Forest et de Régression Linéaire (avec Validation Croisée).
- Découverte majeure : L'ajout de la `Radio` ou de `Social Media` à la `TV` biaise les modèles linéaires (coefficients négatifs aberrants).
- Décision : Conserver UNIQUEMENT la variable `TV` pour prédire les `Sales`.

5. NORMALISATION
--------------------------------------------------------------------------------
- Création du script `normalize_data.py`.
- Application d'un `StandardScaler` pour centrer et réduire (moyenne=0, écart-type=1) les variables `TV` et `Sales`.
- Sauvegarde du jeu de données final prêt pour la modélisation : `data/Normalized_Data_HSS.csv`.

6. MODÉLISATION (MACHINE LEARNING & DEEP LEARNING)
--------------------------------------------------------------------------------
- Création du script d'entraînement `train_models.py` sur les données normalisées.
- Split des données en Train (80%) et Test (20%).
- Entraînement et comparaison de 4 algorithmes :
  * Machine Learning : Régression Linéaire Simple & Support Vector Regression (SVR Linéaire).
  * Deep Learning : Multi-Layer Perceptron (Réseau Profond) & Réseau Linéaire (SGD).
- Résultats exceptionnels : Un R² de 0.999 pour les modèles linéaires (SVR, Régression Linéaire, SGD), confirmant que la TV explique la quasi-totalité des ventes dans cette simulation de données.
- Génération de rapports métriques (R², RMSE, MAE) et graphiques comparatifs dans le dossier `plots/`.

7. CRÉATION DU LIVRABLE FINAL (NOTEBOOK)
--------------------------------------------------------------------------------
- Synthèse de tout ce travail de bout-en-bout (EDA, Nettoyage, VIF, Modélisation) dans un Jupyter Notebook interactif et commenté.
- Fichier généré : `etude_marketing_roi.ipynb` directement à la racine du projet.

--------------------------------------------------------------------------------
Fin du résumé.
